# 预加载策略文档

## 策略概述

本项目实现基于滑动方向的智能预加载策略，在用户观看当前视频时，提前加载前后视频的前 N 字节数据，以减少滑动切换时的等待时间。

## 预加载窗口

- **窗口大小**: 当前视频前后各 2 个（共 5 个视频的窗口）
- **预加载内容**: 每个视频的前 512KB 数据（约 3-5 秒的视频内容）
- **预加载时机**: 当前视频开始播放后立即触发

```
位置:   -2    -1    [当前]   +1    +2
状态:  预加载  预加载   播放   预加载  预加载
```

## 预加载触发条件

1. **滑动切换后**: 用户滑动到新视频时，立即更新预加载窗口
2. **首次加载**: 第一个视频开始播放时，预加载 +1 和 +2 位置的视频
3. **加载更多后**: 新数据追加到列表后，更新预加载窗口

## 资源控制

| 控制项 | 策略 |
|---|---|
| 网络优先级 | 预加载使用低优先级线程池（Thread.MIN_PRIORITY） |
| 线程池大小 | 2 个线程（不与当前播放争抢带宽） |
| 单次预加载量 | 512KB（避免消耗过多流量） |
| 并发控制 | 同一时间最多 2 个预加载任务 |

## 预加载取消策略

- **窗口外取消**: 视频滑出预加载窗口时，立即取消其预加载任务
- **重复检测**: 已预加载的视频不会重复加载
- **正在加载检测**: 正在预加载的视频不会重复启动任务

## 与缓存系统的协作

```
预加载流程:
1. 检查该 URL 是否已缓存 → 跳过
2. 发起 HTTP Range 请求 (bytes=0-524287)
3. 下载的数据写入磁盘缓存
4. 用户滑动到该视频时，播放器从缓存读取首帧
```

## 策略参数（可配置）

| 参数 | 默认值 | 说明 |
|---|---|---|
| PRELOAD_WINDOW | 2 | 前后各预加载 N 个 |
| PRELOAD_BYTES | 512KB | 每个视频预加载的字节数 |
| THREAD_POOL_SIZE | 2 | 预加载线程池大小 |

## 未来优化方向

1. **基于网络状况动态调整**: 弱网时减少预加载数量，强网时增加
2. **基于用户行为预测**: 如果用户频繁快速滑动，增大预加载窗口
3. **预加载完整视频**: WiFi 环境下预加载完整视频而非仅前 N 字节
